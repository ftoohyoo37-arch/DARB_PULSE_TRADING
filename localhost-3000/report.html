Dry-run submit to Crossref test endpoint

Submits the (cleaned, if you have decisions saved) XML to Crossref's test ingestion at https://test.crossref.org/servlet/deposit and shows the raw response. Surfaces business-rule rejections (missing required fields, schema-passing-but-malformed citations) that XSD validation alone misses. Requires a Crossref test account (free at test.crossref.org); credentials are sent per request and not stored.

Important: the test endpoint uses a SEPARATE sandbox database that does not mirror production DOIs. A "Referenced DOI not found in Crossref" record-level failure here may simply mean the parent article DOI exists in production but hasn't been replicated into the test sandbox — not that the DOI is genuinely unregistered. Verify against the Crossref REST API (https://api.crossref.org/works/<DOI>) before treating any record-level failure as a real problem.
